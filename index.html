// optimization engine

One recursive loop

An Archive keeps the top-k agents by score — several, not just the best, so good stepping-stones survive. A Task scores each candidate; a Generator proposes a new agent by mutating top parents; the Loop runs generate → evaluate → select → archive. The twist: the Generator itself is improvable, so the engine can improve how it improves.

↑ the variation operator is itself meta-optimized — recursive adaptation of the search distribution

// empirical result

Self-adaptive σ outperforms a static scale

On a 10-D black-box quadratic — only scalar fitness is observed, never the target — a generator that self-tunes its mutation scale σ via the 1/5th success rule (Rechenberg) outperforms a fixed scale. The same Loop and Archive then transfer unchanged to any domain.
